# ML-M2 Export Contract

## Artifact Roots

Large artifacts are written outside Git:

```text
VEDA_ML_ARTIFACT_ROOT
VEDA_ML_DATA_ROOT
VEDA_HF_CACHE
```

If variables are not set, scripts use a local `build/ml_m2_artifacts/` fallback
for tests only. That path is ignored by Git.

## Exported Groups

Software-side tensors:

- `token_embedding`;
- `position_embedding`;
- `final_norm_gamma`;
- `lm_head`.

RTL layer tensors:

- `norm1_gamma`;
- `wq`;
- `wk`;
- `wv`;
- `wo`;
- `norm2_gamma`;
- `w1`;
- `w2`.

## Per-Tensor Manifest Fields

Each tensor export records:

```text
logical_name
source_state_dict_name
shape
dtype
source_layout
rtl_layout
transpose_applied
element_count
sha256
output_path
```

The RTL weight layout is:

```text
weight[output_index][input_index]
```

PyTorch `Linear.weight` has the same logical direction. Export tests must
confirm this independently.

## Formats

ML-M2 export supports:

- `.npy` for debug and software comparison;
- `.hex` text stream for small RTL fixtures;
- manifest JSON.

Large exports must not be committed. The repository may commit small fixture
exports generated by tests.
